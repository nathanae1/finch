# arti_bridge

C-compatible Rust crate wrapping [Arti](https://gitlab.torproject.org/tpo/core/arti)
for Finch's Flutter app. Surfaces a tiny FFI: bootstrap a `TorClient`,
publish a v3 onion service for the on-device HTTP server, and report
status. See `src/lib.rs` for the C API and `src/inner.rs` for the safe
Rust core.

## Toolchain prerequisites

```sh
# Rust + cross-compile targets
rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios
rustup target add aarch64-linux-android x86_64-linux-android

# Android NDK + cargo-ndk (Android only)
cargo install cargo-ndk
# Set ANDROID_NDK_HOME to your NDK install (e.g. via Android Studio SDK Manager)
export ANDROID_NDK_HOME=$HOME/Library/Android/sdk/ndk/<version>

# iOS only needs Xcode CLT (xcodebuild -create-xcframework)
xcode-select --install
```

## Building

```sh
cd app/finch/native/arti_bridge
./build.sh            # builds iOS + Android
./build.sh ios        # iOS only
./build.sh android    # Android only
./build.sh host       # host target — for cargo check only
```

Outputs land at:
- `app/finch/ios/Runner/arti_bridge.xcframework/` — checked in
- `app/finch/android/app/src/main/jniLibs/{arm64-v8a,x86_64}/libarti_bridge.so` — checked in
- `app/finch/native/arti_bridge/include/arti_bridge.h` — generated by cbindgen, checked in

## Arti versions

`Cargo.toml` pins `arti-client = "0.34"` and matching `tor-hsservice` /
`tor-hsrproxy` / `tor-rtcompat` / `tor-keymgr` / `safelog` versions.
`Cargo.lock` additionally pins `typed-index-collections = 3.2.3` —
later 3.x patches bump MSRV to rustc 1.90 while we ship 1.88; do not
let `cargo update` move that pin without bumping the toolchain.

Arti's onion-service API has moved between releases; if a build picks up
a newer crates.io patch and the call sites in `src/inner.rs` no longer
match, bump the pin and read the Arti
[CHANGELOG](https://gitlab.torproject.org/tpo/core/arti/-/blob/main/CHANGELOG.md).

The known-fragile call sites:
- `OnionServiceConfigBuilder` — field set + builder location
- `TorClient::launch_onion_service` — return shape
- `OnionServiceReverseProxy::handle_requests` — argument list
- `BootstrapStatus::as_frac` / `ready_for_traffic`

## Binary size budget

Plan 11 budgets 10–20 MB. Reference numbers from a clean release build
on macOS arm64 host:

| Artifact                     | Size  |
|------------------------------|-------|
| `libarti_bridge.a` (static)  | 13 MB |
| `libarti_bridge.dylib`       | 3.3 MB|

iOS uses the static slice (xcframework). Android uses dynamic .so per
arch. Expect Android .so at ~3–5 MB per arch and the iOS binary
contribution to land around 13 MB after the linker dead-strips.

If over budget:
- Drop `arti-client` features `pt-client`, `experimental-api`, `geoip`.
- Strip with `cargo build --release` already enabled (`strip = "symbols"`
  in `[profile.release]`).
- Consider `tor-rtcompat`'s `rustls` feature instead of `native-tls`.

## SOCKS proxy

11a returns 0 from `arti_socks_port`. Plan 11b will run Arti's SOCKS
proxy in-process so the Dart `http.Client` can route through Tor.
